# NFTables 协议转发指南

## 🔍 NFTables 协议转发限制

### NFTables NAT 的技术约束

NFTables 在网络地址转换（NAT）层面存在协议版本限制：

```bash
# ✅ 支持的转发方式
IPv4 → IPv4 转发    # table ip nat
IPv6 → IPv6 转发    # table ip6 nat

# ❌ 不支持的转发方式
IPv4 → IPv6 转发    # 需要协议转换
IPv6 → IPv4 转发    # 需要协议转换
```

### 为什么不能跨协议转发？

1. **数据包结构差异**

   - IPv4: 32 位地址，特定头部格式
   - IPv6: 128 位地址，不同头部结构

2. **NAT 表限制**

   - `table ip nat` 只处理 IPv4 数据包
   - `table ip6 nat` 只处理 IPv6 数据包

3. **内核处理机制**
   - netfilter hooks 按协议版本分离处理

### 实现优势

1. **兼容性最佳**: 相同协议版本转发

4. **性能最优**: 无协议转换开销
