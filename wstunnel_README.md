# wstunnel 管理脚本使用指南

## 简介

这是一个简洁高效的 [wstunnel](https://github.com/erebe/wstunnel) 客户端管理脚本，提供了直观的菜单界面和智能默认值设置。

## 功能特性

- 🚀 **智能默认值** - 回车即可使用推荐配置
- 🎨 **简洁界面** - 仅使用三种颜色，清晰易读
- 🔀 **一键双协议** - 默认同时创建TCP+UDP转发
- � **自动端口** - 支持随机分配可用端口
- 🌐 **单一API** - 快速获取完整IP信息
- 📝 **自动备注** - 智能生成规则描述

## 快速使用

### 安装运行

```bash
# 一键安装运行
bash <(curl -Ls https://raw.githubusercontent.com/ilychi/snells/main/wstunnel.sh)

# 或下载后运行
wget https://raw.githubusercontent.com/ilychi/snells/main/wstunnel.sh
chmod +x wstunnel.sh
sudo ./wstunnel.sh
```

### 界面说明

脚本启动后显示：
- **服务状态** - 运行状态、内存使用、活跃规则数
- **IP信息** - IPv4/IPv6、地理位置、ISP、ASN
- **快捷菜单** - 数字键快速选择功能

## 快速操作指南

### 添加转发（最简单）

1. 选择 `1` 添加规则
2. **协议**：直接回车（默认TCP+UDP）
3. **监听**：直接回车（默认IPv4）
4. **端口**：直接回车（自动分配）
5. **目标**：输入目标地址（必填）
6. **目标端口**：回车使用相同端口
7. **备注**：回车自动生成

### 常用场景示例

#### 1. Web服务转发（HTTP/HTTPS）
```
协议: 回车 (TCP+UDP)
端口: 80
目标: internal.server.com
目标端口: 回车 (80)
备注: 回车 (自动)
```

#### 2. WireGuard VPN
```
协议: 3 (TCP+UDP)
端口: 51820
目标: localhost
目标端口: 回车 (51820)
UDP超时: 回车 (0-不超时)
```

#### 3. SOCKS5代理
```
协议: 4 (SOCKS5)
端口: 回车 (随机)
备注: 回车 (自动)
```

#### 4. 游戏服务器
```
协议: 回车 (TCP+UDP)
端口: 25565
目标: mc.server.local
目标端口: 回车 (25565)
```

## 默认值说明

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 协议 | TCP+UDP | 同时转发两种协议 |
| 监听地址 | 0.0.0.0 | IPv4全部接口 |
| 本地端口 | 随机 | 49152-65535范围 |
| 目标端口 | 同本地端口 | 简化配置 |
| UDP超时 | 0 | 永不超时 |
| 备注 | 自动生成 | 格式：协议-端口→目标 |

## 菜单快捷键

### 转发管理
- `1` - 添加规则
- `2` - 查看规则（●启用 ○禁用）
- `3` - 删除规则
- `4` - 启用/禁用切换

### 服务管理
- `5` - 配置服务器
- `6` - 启动服务
- `7` - 停止服务
- `8` - 重启服务
- `9` - 查看日志

### 系统管理
- `a` - 更新wstunnel
- `b` - 开机启动切换
- `0` - 退出

## 配置文件

配置保存在 `/etc/wstunnel/config.json`：

```json
{
  "server": {
    "url": "wss://your-server.com",
    "secret": "optional-secret"
  },
  "tunnels": [
    {
      "id": "abc12345-tcp",
      "protocol": "tcp",
      "listen_addr": "0.0.0.0",
      "local_port": "8080",
      "remote_host": "example.com",
      "remote_port": "80",
      "comment": "TCP+UDP-8080→example.com:80 (TCP)",
      "enabled": true
    }
  ]
}
```

## 高级技巧

### 批量端口转发
利用自动端口功能快速创建多个转发：
1. 连续按 `1` 添加规则
2. 每次都按回车使用随机端口
3. 只需输入不同的目标地址

### 临时禁用规则
无需删除，使用 `4` 切换启用/禁用状态

### 查看实时日志
选择 `9` 查看最近30条日志，快速排查问题

## 故障排查

### 服务无法启动
1. 检查是否配置服务器（选项5）
2. 确认服务器地址格式：`wss://domain.com`
3. 查看日志获取详细错误

### 端口被占用
- 使用自动端口分配（直接回车）
- 或手动指定其他端口

### 转发不工作
1. 确认规则已启用（绿色●）
2. 检查防火墙设置
3. 验证目标服务可访问

## 性能优化

- **内存监控**：主界面显示内存使用
- **规则管理**：及时禁用不用的规则
- **日志清理**：定期清理系统日志

## 安全建议

1. **使用WSS协议**：始终使用加密连接
2. **设置密钥前缀**：增加额外安全层
3. **限制监听地址**：仅在需要时使用0.0.0.0
4. **定期更新**：使用 `a` 键更新程序
