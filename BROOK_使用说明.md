# Brook 端口转发管理脚本使用说明

## 简介

Brook 端口转发管理脚本是一个功能强大的自动化工具，支持 TCP、UDP 以及混合协议的端口转发管理。该脚本提供了友好的交互界面，支持自动安装、配置管理、服务监控等功能。

## 功能特点

1. **自动安装**: 自动检测并安装所需依赖和 Brook 程序
2. **多协议支持**:
   - 仅 TCP 转发
   - 仅 UDP 转发
   - TCP+UDP 同时转发到相同地址
   - TCP 和 UDP 分别转发到不同地址
3. **服务管理**: 使用 systemd 管理转发服务，支持开机自启
4. **可视化管理**: 提供友好的菜单界面，易于操作
5. **日志查看**: 方便查看服务运行日志，便于故障排查

## 安装使用

## 使用说明

### 主菜单功能

运行脚本后，会显示以下菜单：

```
========== Brook 端口转发管理 ==========
  1. 添加转发
  2. 列出所有转发
  3. 删除转发
  4. 重启所有服务
  5. 查看服务日志
  6. 卸载Brook
  0. 退出
=======================================
```

### 1. 添加转发

选择添加转发后，会提供 4 种转发模式：

#### 模式 1：仅 TCP 转发

- 适用场景：只需要转发 TCP 流量
- 配置示例：本地 8080 端口 -> 远程服务器 192.168.1.100:80

#### 模式 2：仅 UDP 转发

- 适用场景：只需要转发 UDP 流量（如 DNS、游戏等）
- 配置示例：本地 53 端口 -> 远程 DNS 服务器 8.8.8.8:53

#### 模式 3：TCP+UDP 转发到相同地址

- 适用场景：需要同时转发 TCP 和 UDP 到同一个目标
- 配置示例：本地 3000 端口 -> 远程服务器 10.0.0.1:3000（TCP 和 UDP 都转发）

#### 模式 4：TCP 和 UDP 分别转发到不同地址

- 适用场景：需要将 TCP 和 UDP 流量转发到不同的目标
- 配置示例：
  - TCP: 本地 5000 端口 -> 服务器 A 192.168.1.10:5000
  - UDP: 本地 5000 端口 -> 服务器 B 192.168.1.20:5000

### 2. 列出所有转发

显示当前所有活动的转发服务，包括：

- 服务名称
- 本地监听端口
- 目标地址
- 协议类型（TCP/UDP/both）
- 运行状态

### 3. 删除转发

- 先列出所有转发服务
- 输入要删除的服务编号
- 确认后删除对应的转发服务

### 4. 重启所有服务

一键重启所有 Brook 转发服务，用于配置更新后生效。

### 5. 查看服务日志

- 可以查看单个服务的日志
- 也可以查看所有服务的日志
- 显示最近 50 条日志记录

### 6. 卸载 Brook

完全卸载 Brook 程序和所有转发服务。

## 配置文件位置

- 配置目录：`/etc/brook/`
- 配置文件：`/etc/brook/forwards.conf`
- 服务文件：`/etc/systemd/system/brook-forward-*.service`

## 技术细节

### Brook relay 参数说明

- `--tcpTimeout 0`: 设置为 0 表示禁用 TCP 转发
- `--udpTimeout 0`: 设置为 0 表示禁用 UDP 转发
- 不设置或设置为正数：表示超时时间（秒）

### systemd 服务命名规则

服务名格式：`brook-forward-[本地端口]-[协议]`

例如：

- `brook-forward-8080-tcp`: 8080 端口的 TCP 转发服务
- `brook-forward-53-udp`: 53 端口的 UDP 转发服务
- `brook-forward-3000-both`: 3000 端口的 TCP+UDP 转发服务

## 常见问题

### 1. 端口被占用

错误提示：端口已被占用
解决方法：

```bash
# 查看端口占用情况
lsof -i:端口号
# 或
netstat -tlnp | grep 端口号
```

### 2. 服务启动失败

查看服务状态和日志：

```bash
# 查看服务状态
systemctl status brook-forward-端口-协议

# 查看详细日志
journalctl -u brook-forward-端口-协议 -n 50
```

### 3. 权限问题

如果遇到权限相关错误，确保：

- 使用 sudo 运行需要管理员权限的操作
- 配置文件权限正确（644）
- systemd 服务文件权限正确（644）

### 4. 防火墙配置

确保防火墙允许相应端口：

```bash
# firewalld
sudo firewall-cmd --permanent --add-port=端口号/tcp
sudo firewall-cmd --permanent --add-port=端口号/udp
sudo firewall-cmd --reload

# ufw
sudo ufw allow 端口号/tcp
sudo ufw allow 端口号/udp
```

## 使用示例

```
选择：4（TCP和UDP分别转发）
本地端口：25565
TCP目标：游戏服务器1:25565
UDP目标：游戏服务器2:25565
```

```
选择：2（仅UDP转发）
本地端口：53
目标地址：8.8.8.8:53
```

### 示例 3：Web 服务转发

```
选择：1（仅TCP转发）
本地端口：80
目标地址：内网服务器:8080
```

## 注意事项

1. **端口范围**：支持 1-65535 的所有端口
2. **目标地址**：支持 IPv4、IPv6 和域名
3. **服务持久化**：所有转发服务都会在系统重启后自动启动
4. **资源占用**：每个转发服务占用少量内存，大量转发时请注意系统资源
